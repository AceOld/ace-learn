Part 1 · 数据类陷阱(4 个)

数据问题是最隐蔽的陷阱 —— 代码完全正确,但数据本身就让结果失真。这类陷阱最难发现,因为你不会怀疑"数据有问题",只会怀疑"策略有问题"。

陷阱 1 · 存活偏差(Survivorship Bias)

定义:用当前的股票池回测历史,排除了期间退市/被摘牌的公司。结果:只测了"活下来的",而这些常常是表现好的,回测收益被系统性高估。

你今天遇到过的真实例子

Q·09 的 fetch 脚本用 stock_info_a_code_name 拉"当前在市股票"做 2019-2021 回测。但 2019 年在市、2024 年退市的股票(常常是表现差的)不在样本里 → 策略表现被高估。

复现代码

"""复现存活偏差的影响"""
import akshare as ak
import polars as pl
import numpy as np

# 当前在市股票(A股全市场)
current = ak.stock_info_a_code_name()
print(f"当前在市:{len(current)} 只")

# 模拟:假设我们要回测 2019-2021 的"全市场等权"策略
# 如果用当前股票池(有存活偏差),vs 用 2019 年初的真实股票池(无偏差)
# 由于 AKShare 不直接给历史股票池,这里用估算说明

# 真实情况(2019-2024 退市统计,数据来自 wind 整理):
# - 2019 年初在市约 3500 只
# - 2024 年在市约 5000 只(+1500 新上市)
# - 期间退市约 200 只(强制退市 + 重组)
# - 退市股平均跌幅:-70%(退市前一年)

n_current = 5000      # 当前股票池
n_delisted = 200      # 期间退市
delisted_avg_return = -0.70

# 等权"全市场"组合的真实收益(假设当前股票池平均 +30%)
true_avg_return_current_pool = 0.30
total_market_cap_gains = n_current * true_avg_return_current_pool

# 退市股的贡献(被存活偏差排除掉)
delisted_contribution = n_delisted * delisted_avg_return

# 无偏差的真实平均收益
true_avg_with_delisted = (total_market_cap_gains + delisted_contribution) / (n_current + n_delisted)

print(f"\n=== 存活偏差影响估算 ===")
print(f"用当前股票池回测平均收益:{true_avg_return_current_pool:.2%}")
print(f"加入退市股后的真实收益:{true_avg_with_delisted:.2%}")
print(f"存活偏差导致的高估      :{true_avg_return_current_pool - true_avg_with_delisted:.2%}")
print(f"\n结论:仅 200 只退市股就让回测高估 {(true_avg_return_current_pool - true_avg_with_delisted)*100:.1f} 个百分点")
典型输出
用当前股票池回测平均收益:30.00%
加入退市股后的真实收益:27.06%
存活偏差导致的高估      :2.94%

结论:仅 200 只退市股就让回测高估 2.9 个百分点

修复方法

# 用动态成分股(每个时点用当时的真实股票池)
# AKShare 不直接提供,用 Tushare 或 wind 数据
# Tushare 示例(需积分):
import tushare as ts
pro = ts.pro_api("your_token")

# 拉 2019 年初的真实股票池
pool_2019 = pro.stock_basic(exchange='', list_status='L',
                             fields='ts_code,symbol,name,list_date,delist_date')
# 过滤:2019-01-01 时在市(list_date <= 2019-01-01,delist_date IS NULL 或 > 2019-01-01)
pool_2019 = pool_2019[
    (pool_2019['list_date'] <= '20190101') &
    ((pool_2019['delist_date'].isna()) | (pool_2019['delist_date'] > '20190101'))
]
print(f"2019-01-01 真实在市:{len(pool_2019)} 只(含后来退市的)")

✅ 回测开始日的股票池是否包含当时在市的所有股票(包括后来退市的)?
✅ 数据源是否提供 list_date 和 delist_date?
✅ 是否使用"动态成分股"而非"固定当前成分股"?

陷阱 2 · 前视偏差(Look-Ahead Bias)

定义:回测时使用了决策时刻还不知道的信息。最常见:用当日收盘价决定当日买卖 → 但收盘时你才知道价格,实际下单是次日开盘。

你今天测过的真实例子

在 squeeze_biases.py 里测过:横截面动量策略用"当日收盘价"成交 vs "次日开盘价"成交,2019-2021 收益从 269% 降到 208%(缩水 60 个点)。60 个百分点就是前视偏差的代价。

复现代码

"""复现前视偏差"""
import polars as pl
import pandas as pd
import numpy as np

# 加载你今天的数据
prices = pl.read_parquet("./data/hs300_2025_hfq.parquet")
wide = prices.select(["date","symbol","close"]).pivot(on="symbol", index="date", values="close").sort("date")
price_df = wide.drop("date").to_pandas().ffill().bfill()

# 假设策略:每月末选过去20天 top 5 涨幅,买入持有1月
# 对比:当日收盘价成交 vs 次日开盘价成交(用 open 数据更真实,这里简化用 close 模拟)

def backtest_with_lookahead(price_df, use_next_day=True):
    """use_next_day=False = 有前视(当日成交);True = 无前视(次日成交)"""
    dates = pd.to_datetime(wide["date"].to_list())
    month_end = pd.Series(dates).groupby([pd.Series(dates).dt.year,
                                          pd.Series(dates).dt.month]).apply(lambda x: x.index[-1]).tolist()

    cash = 1_000_000
    holdings = {}
    values = []

    for i in range(len(dates)):
        today_price = price_df.iloc[i]

        # 调仓决定日
        signal_day = i - 1 if use_next_day else i
        if signal_day in month_end and signal_day >= 20:
            past = price_df.iloc[signal_day - 20]
            mom = (today_price / past - 1).dropna() if use_next_day else \
                  (price_df.iloc[signal_day] / past - 1).dropna()
            top5 = mom.nlargest(5).index

            # 卖出
            cash += sum(s * today_price.get(s, 0) for s, sz in holdings.items() if s not in top5) * 0.999
            holdings = {s: sz for s, sz in holdings.items() if s in top5}

            # 买入
            per = cash / 5
            for s in top5:
                px = today_price.get(s, 0)
                if px > 0:
                    shares = int(per / px / 100) * 100
                    if shares > 0:
                        cash -= shares * px * 1.001
                        holdings[s] = shares

        values.append(cash + sum(sz * today_price.get(s, 0) for s, sz in holdings.items()))

    return (values[-1] / 1_000_000 - 1)

ret_lookahead = backtest_with_lookahead(price_df, use_next_day=False)
ret_no_lookahead = backtest_with_lookahead(price_df, use_next_day=True)

print(f"=== 前视偏差影响(2025 数据,top5 月度调仓)===")
print(f"有前视(当日收盘成交):{ret_lookahead:.2%}")
print(f"无前视(次日开盘成交):{ret_no_lookahead:.2%}")
print(f"前视偏差虚高        :{(ret_lookahead - ret_no_lookahead)*100:.1f} 个百分点")

修复方法

# 规则:决策和执行分开
# T 日收盘后:计算信号(基于 T 日收盘价)
# T+1 日开盘:执行交易(用 T+1 日开盘价)
#
# 代码层面:
# - 信号计算用 price_df.iloc[t]
# - 实际成交用 price_df.iloc[t + 1] 的 open 列
# - 千万不要用 close 列同时做信号和成交

# vectorbt 的正确姿势:
pf = vbt.Portfolio.from_signals(
    price_df,
    entries=signals,            # 在 t 日生成
    open=next_day_open,         # 显式指定次日开盘成交
    ...
)
检查清单

✅ 信号生成和成交是否使用不同时点的价格?
✅ 当日信号是否在次日(或更晚)才执行?
✅ 用 vbt.Portfolio.from_signals(..., open=next_open) 时是否显式指定 open 列?

Q1 · 你在月末当日用收盘价计算信号,又用同一收盘价"成交"。这算什么偏差?
  • 存活偏差
  • 前视偏差
  • 参数过拟合
  • 没有偏差,合理
答案:B。收盘时你才知道收盘价,但回测却假设你能在收盘那一瞬间以该价格成交 —— 实际你只能以次日开盘成交(或当晚挂单,但成交价不确定)。这就是经典前视偏差。修复:信号用 t 日收盘,执行用 t+1 日开盘。A(存活偏差)是用当前股票池测历史;C(过拟合)是参数太多。今天你的 squeeze_biases 测过,2019-2021 的 60 个百分点差距就是它。

陷阱 3 · 未来函数(Future Function)

定义:用了决策时点之后才发布的数据。和前视偏差类似,但更隐蔽 —— 通常发生在财务数据/宏观数据上,因为发布有滞后。

典型场景

Q1 财报通常在 4 月底才公布。但有些数据源(为了方便)会把 Q1 财务数据时间戳到 3 月 31 日(报告期末)。你的回测代码 4 月 1 日就读到"Q1 EPS",但其实那时 Q1 财报还没公布 → 未来函数。

复现代码

"""复现未来函数(财报时间戳陷阱)"""
import pandas as pd
import numpy as np

# 模拟:某公司 2024 Q1 EPS = 1.5 元,实际 2024-04-30 才公布
# 数据源错把 announce_date 标成 2024-03-31(报告期末)
earnings = pd.DataFrame({
    "report_date": ["2024-03-31", "2024-06-30", "2024-09-30", "2024-12-31"],
    "announce_date_wrong": ["2024-03-31", "2024-06-30", "2024-09-30", "2024-12-31"],  # 错!
    "announce_date_correct": ["2024-04-30", "2024-08-31", "2024-10-31", "2025-04-30"],  # 对
    "eps": [1.5, 1.7, 1.6, 2.0],
})

# 错误策略:在 report_date 当日就用 EPS 选股
# 结果:你"提前"知道了 1-3 个月的财报数据
print("=== 错误(未来函数)===")
print("策略 4 月 1 日就用了 Q1 EPS,实际 4 月 30 日才公布")
print("你比市场早知道了 30 天 → 这 30 天的超额收益是假的\n")

# 正确:在 announce_date 当日才能用
print("=== 正确(无未来函数)===")
print("策略 4 月 30 日(公布日)才用 Q1 EPS")
print("和市场同步,这才是真实信号")

修复方法

# 财报数据必须用 announce_date(公布日),不是 report_date(报告期末)
# AKShare / Tushare 都提供 announce_date,关键是你用对没有

# 正确的财报回测:
df = pro.income(ts_code='600000.SH', period='20240331')
# 必须等到 ann_date(公布日)之后才能用
# 不能用 end_date(报告期末)

# 通用规则:任何宏观数据(CPI/PMI/GDP)都要用公布日,
# 公布日通常比报告期晚 1-4 周
检查清单

✅ 财报数据用的是 ann_date(公布日)还是 end_date(期末)?
✅ 宏观数据(CPI/PMI)是否用公布日而非报告期?
✅ 任何"当日知道当日数据"的假设都需怀疑,大部分宏观数据有 1-30 天滞后。

陷阱 4 · 复权错误(Adjustment Error)

定义:用错复权方式让历史价格"变形"。最常见:跨日期拼接前复权(qfq)数据 → 分红后整段历史价格都变了 → 拼接处出现幻觉跳空。

Q·09 讲过的经典陷阱

前复权(qfq)锚定"今天"的最新复权因子。你周一拉了 2023 年 qfq,周三该公司分红后,你再拉 2023 年 qfq 会得到完全不同的数字。跨日期拼接 = 幻觉跳空/涨跌停。

复现代码

"""复现复权时变性"""
import akshare as ak

# 同一只股票,不同日期拉的 qfq 历史价格会不同
# (假设公司在此期间分红了)
code = "000001"
print(f"=== {code} 复权方式对比 ===\n")

# 1. 不复权(raw)
raw = ak.stock_zh_a_hist(symbol=code, start_date="20240101", end_date="20240105", adjust="")
print(f"不复权:    {raw['收盘'].iloc[0]:.2f} → {raw['收盘'].iloc[-1]:.2f}")

# 2. 前复权(qfq)- 锚定"今天",分红后会变
qfq = ak.stock_zh_a_hist(symbol=code, start_date="20240101", end_date="20240105", adjust="qfq")
print(f"前复权:    {qfq['收盘'].iloc[0]:.2f} → {qfq['收盘'].iloc[-1]:.2f}")

# 3. 后复权(hfq)- 锚定 IPO 日,历史稳定
hfq = ak.stock_zh_a_hist(symbol=code, start_date="20240101", end_date="20240105", adjust="hfq")
print(f"后复权:    {hfq['收盘'].iloc[0]:.2f} → {hfq['收盘'].iloc[-1]:.2f}")

print(f"""
=== 关键认知 ===
- 不复权:除权日会跳空(假跌),回测收益错误
- 前复权:历史数据随时间变化(分红后会重算),跨日期拼接有 bug
- 后复权:历史数据稳定(锚定 IPO 日),适合回测 ← 推荐
- 最佳:存不复权 + 复权因子,按需现算 hfq
""")

修复方法

# 三选一(从优到劣):
#
# 方案 A(最佳):存"不复权 + 复权因子"
#   /data/raw/date=.../data.parquet       # 原始价格
#   /data/adj_factor/date=.../data.parquet # 复权因子
#   回测时:hfq_close = raw_close * cum_adj_factor

# 方案 B(简单):直接拉 hfq 一次拉全,不要跨日期拼接
#   优点:数据稳定
#   缺点:无法回算 qfq

# 方案 C(禁用):跨日期拉 qfq 拼接
#   必然有幻觉跳空,绝对不要这样做
检查清单

✅ 回测用的是 hfq(后复权)还是 qfq(前复权)?
✅ 如果是 qfq,是否在同一天一次性拉全?
✅ 除权除息日的收益计算是否合理(不复权会跳空)?

Q2 · 你今天用 hfq 拉的 2025 数据,在 2027 年再用同代码拉同股票,价格会?
  • 完全不同(锚定今天,会变)
  • 部分不同(分红后才变)
  • 相同(hfq 锚定 IPO 日,历史稳定)
  • 不能确定
答案:C。后复权(hfq)锚定 IPO 日,历史价格只取决于"IPO 到当时的累计复权因子",与"今天"无关。所以 2027 年再拉 2025 年的 hfq 数据,结果和今天拉的完全一致(除非数据源修订历史复权因子,极少见)。这是为什么 Q·09 强烈推荐 hfq 用于回测。A 描述的是 qfq(前复权);B 是 qfq 的部分情形;D 是不确定。

Part 2 · 策略类陷阱(3 个)

策略类陷阱与数据无关,是策略设计/选择本身导致的失真。这类陷阱最常见,也最容易让人产生"我发现了 alpha"的幻觉。

陷阱 5 · 样本不完整(Incomplete Sample)

定义:回测数据缺股票或缺时间段,导致样本不具代表性。最隐蔽的陷阱之一 —— 代码完全正确,但数据不全让结论反转。

你今天亲历的真实案例

拉沪深 300 时被东财封,只拉到 133 只 → SMA20/50 +16.61% / 75% 胜率。补齐到 298 只后 → -34.12% / 26% 胜率。同一策略同一参数,结论完全反转。这就是样本不全的威力。

复现代码(基于你今天的数据)

"""复现样本不完整的影响"""
import polars as pl

# 完整数据
full = pl.read_parquet("./data/hs300_prices_hfq.parquet")
print(f"完整样本:{full['symbol'].n_unique()} 只")

# 模拟"只拉到 133 只"(你今天的真实情况)
import random
random.seed(42)
all_syms = full['symbol'].unique().to_list()
partial_syms = set(random.sample(all_syms, 133))   # 随机抽 133 只
partial = full.filter(pl.col("symbol").is_in(partial_syms))

# 简单计算"全市场等权"组合的总收益(每月等权买入持有)
def equal_weight_return(df):
    # 每只股票的总收益
    per_stock = df.sort(["symbol", "date"]).group_by("symbol").agg(
        first_close=pl.col("close").first(),
        last_close=pl.col("close").last(),
    )
    per_stock = per_stock.with_columns(
        (pl.col("last_close") / pl.col("first_close") - 1).alias("return")
    )
    return per_stock["return"].mean()

full_ret = equal_weight_return(full)
partial_ret = equal_weight_return(partial)

print(f"\n=== 等权组合 2022-2024 收益 ===")
print(f"完整 298 只:{full_ret:.2%}")
print(f"部分 133 只:{partial_ret:.2%}")
print(f"差距:{(partial_ret - full_ret)*100:.1f} 个百分点(样本不全的偏差)")

修复方法

# 1. 永远在 fetch 脚本里加"完整性检查"
def check_completeness(df, expected_n=300, expected_days=730):
    actual_n = df["symbol"].n_unique()
    actual_days = df["date"].n_unique()
    if actual_n < expected_n * 0.95:
        print(f"⚠ 警告:股票数 {actual_n} < 预期 {expected_n} 的 95%")
    if actual_days < expected_days * 0.95:
        print(f"⚠ 警告:交易日 {actual_days} < 预期 {expected_days} 的 95%")
    return actual_n >= expected_n * 0.95 and actual_days >= expected_days * 0.95

# 2. 拉数据失败要重试或换源(像今天用 BaoStock 补 AKShare)
# 3. 关键回测前先打印数据画像(n stocks, n days, n nulls)

陷阱 6 · 参数过拟合(Parameter Overfitting)

定义:测了很多参数组合,选最好的那个 → 实际是选了巧合,不是真 alpha。参数越多、测得越多,过拟合越严重。

你今天的真实案例

测了 4 个 SMA 组合(5/20, 10/30, 20/50, 50/200),选了 10/30(+29.97%)。但 测的越多越容易找到"看似好"的组合,这是过拟合。后来横截面动量的参数敏感性测试(12 个组合 11 个夏普>1.2)才是反过拟合证据。

复现代码(参数过拟合模拟)

"""复现参数过拟合:测 100 个随机参数,选最好"""
import numpy as np
import pandas as pd

np.random.seed(0)

# 模拟:假设策略完全无效(纯随机),夏普服从 N(0, 0.3)
# 你测 100 个参数组合,选夏普最高的
n_params = 100
true_sharpe_distribution = np.random.normal(0, 0.3, n_params)

print(f"=== {n_params} 个随机参数(策略其实无效)===")

# 2. Walk-Forward 分析(滚动样本外)
# 每次用前 N 个月选参数,后 M 个月验证,滚动重复
from itertools import product
def walk_forward(price_df, lookback_window=252, test_window=63):
    """每 252 天训练,63 天测试"""
    results = []
    for start in range(0, len(price_df) - lookback_window - test_window, test_window):
        train = price_df.iloc[start : start + lookback_window]
        test = price_df.iloc[start + lookback_window : start + lookback_window + test_window]
        # 在 train 上选最优参数
        # 在 test 上验证
        results.append((train, test))
    return results

# 比如原本 p < 0.05 显著,测了 10 个参数 → p < 0.005 才算显著

陷阱 8 · 杠杆 Bug(隐性借贷)

定义:回测框架允许"借款买股",让持仓 > 本金。结果把"加杠杆偶然赚的钱"算成"策略收益"。这是今天最戏剧化的 bug。

你今天亲历的真实案例

import vectorbt as vbt
import pandas as pd

    price_df, entries, exits,
    init_cash=1_000_000, freq="D",
    cash_sharing=True, group_by=True,
    # 没有约束 size!
)
print(f"❌ 错误(默认):{pf_bad.total_return():.2%}")

# 正确(约束每次买入用现金的 1/n)
pf_good = vbt.Portfolio.from_signals(
    price_df, entries, exits,
    init_cash=1_000_000, freq="D",
    cash_sharing=True, group_by=True,
    size=1.0 / n_stocks,         # 每只 1/n 的现金
    size_type="percent",
    accumulate=False,
)
print(f"\n✅ 正确(无杠杆):{pf_good.total_return():.2%}")
print(f"   持仓峰值:¥{pf_good.asset_value(group_by=True).max():,.0f}")

修复方法

# 如果你不想加杠杆,max_leverage = 1.0

# 几乎肯定有杠杆 bug

# 简单的夏普置信区间估计(Lo 2002 公式):
def sharpe_ci(sharpe, n_periods, confidence=0.95):
    """夏普的置信区间"""
    from scipy import stats
    z = stats.norm.ppf(1 - (1 - confidence) / 2)
    se = np.sqrt((1 + sharpe**2 / 2) / n_periods)
    return sharpe - z * se, sharpe + z * se

陷阱 11 · 多重比较谬误(Multiple Testing)

定义:测了很多策略/因子,选最好的 → 即使所有策略都无效,也会有几个"显著"。这是过拟合的统计版本。

为什么 Harvey《Backtesting》强调

假设你测了 100 个因子,5% 显著性 → 即使全部无效,平均也会有 5 个"假阳性"。这是量化研究的系统性偏差。Harvey & Liu(2014)估计:已发表的金融因子有大部分是假阳性。

复现代码(多重比较)

"""复现多重比较谬误"""
import numpy as np
from scipy import stats

np.random.seed(42)

# 模拟:测 100 个"完全无效"的因子(随机噪声)
n_factors = 100
n_obs = 60  # 60 个月数据

significant_count = 0
for i in range(n_factors):
    # 因子值 = 随机噪声
    factor = np.random.normal(0, 1, n_obs)
    # 收益 = 随机噪声(与因子无关)
    returns = np.random.normal(0, 0.05, n_obs)

    # 回归,看 p 值
    from numpy.polynomial import polynomial as P
    slope = np.cov(factor, returns)[0, 1] / np.var(factor)
    # t 检验
    residuals = returns - slope * factor
    se = np.sqrt(np.var(residuals) / (np.var(factor) * n_obs))
    t_stat = slope / se if se > 0 else 0
    p_value = 2 * (1 - stats.norm.cdf(abs(t_stat)))

    if p_value < 0.05:  # 5% 显著
        significant_count += 1

print(f"=== 测 {n_factors} 个无效因子 ===")
print(f"理论上应该有 ~{n_factors * 0.05:.0f} 个假阳性")
print(f"实际找到的'显著'因子:{significant_count} 个")
print(f"\n这就是多重比较谬误:即使全部因子无效,也能找到 {significant_count} 个'显著'")
print(f"如果你不知道测了多少次,会以为这些因子有效")

修复方法

threshold = 0.05 / n_tests

# 3. Benjamini-Hochberg(控制 FDR,假发现率)

  • 完全可信,数字就是数字
  • 不可信,样本太小(需至少 30 个观测)
  • 完全不可信,必然是 bug
  • 看绝对值,不看样本
答案:B。11 个月 = 11 个观测,远低于统计显著的 30 个门槛。即使策略真实夏普 = 0,小样本偶然也能跑出 2.0+。小样本上看到高夏普,要先存疑而不是庆祝。修复:① 增加样本(至少 3 年)② 蒙特卡洛模拟夏普分布 ③ 报告置信区间。A 是数字崇拜;C 太极端(可能是真的);D 错(样本量决定可信度)。

总结 · 回测陷阱检查清单(打印这一页)

未来每个新策略上线前,过一遍这张表。任何一项不过,都不要上实盘。

#陷阱类别核心检查
1存活偏差数据是否用动态成分股(含已退市)?
2前视偏差数据信号和成交是否用不同时点?
3未来函数数据财报用 ann_date 而非 end_date?
4复权错误数据用 hfq 而非跨日期拼接的 qfq?
5样本不完整策略数据完整性是否 ≥95%?
6参数过拟合策略是否样本外验证 + 参数敏感性测试?
7市场状态依赖策略是否测过极端事件(股灾/疫情)?
8杠杆 Bug执行持仓峰值 ≤ 本金 ×(1+杠杆)?
9滑点/流动性执行是否过滤低流动性股 + 建模滑点?
10小样本偶然统计样本数 ≥30?
11多重比较统计测了多少次?做了 Bonferroni 校正?
12Regime 过拟合统计是否测过多个 regime?

本课要点回顾

  1. 任何回测的好结果都先假设是错的,直到排除所有陷阱。
  2. 数据类陷阱最隐蔽:代码对但数据本身错(存活/前视/未来函数/复权)。
  3. 策略类陷阱最常见:过拟合 / 样本不全 / 单 regime。
  4. 执行类陷阱最致命:杠杆 bug / 滑点,实盘直接打折。
  5. 统计类陷阱最微妙:小样本偶然 + 多重比较,需要统计训练才能识别。
  6. 职业量化的 80% 时间不是设计策略,是排除陷阱。
