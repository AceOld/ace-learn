时间序列:ARIMA + GARCH

前 6 课都是静态/横截面视角。L02 预告过"波动率聚类"需 GARCH,本课兑现承诺 —— 正式进入时间序列建模。

节奏:约 45 分钟

L01-L06 我们把收益当成 i.i.d. 抽样(独立同分布)。但金融数据有时间结构:收益本身接近无自相关(弱有效市场),但波动率有强聚类(L02 已实证)。这一课用两个模型分别处理这两个事实:ARIMA 刻画均值自相关,GARCH 刻画方差聚类。

本课产出
  1. 理解 ARIMA(p,d,q) 的含义和定阶
  2. 推导 GARCH(1,1) 并理解它如何捕捉波动率聚类
  3. Python 用 arch 库跑 GARCH,提取条件波动率
  4. 看懂为什么"GARCH 比 L02 的历史窗口波动率更现实"

1. ARIMA:建模均值自相关

ARIMA(p, d, q) 三个参数:

  • p:AR(自回归)阶数 —— $r_t$ 依赖过去 $p$ 期
  • d:差分阶数 —— 把非平稳序列变平稳
  • q:MA(滑动平均)阶数 —— $r_t$ 依赖过去 $q$ 期冲击

AR(p) 模型:

$$r_t = c + \phi_1 r_{t-1} + \phi_2 r_{t-2} + \cdots + \phi_p r_{t-p} + \varepsilon_t$$

⚠ 金融收益的 ARIMA 通常"什么都不要"

对日度对数收益,$d=1$(价格 $P_t$ 是 I(1),收益 $r_t$ 已平稳),且 p=q=0 是常态(收益无自相关,弱有效市场)。也就是说 ARIMA(0,1,0) 就够。如果你的 AR 模型在收益上挖出强显著的 $\phi$,要么是数据 bug,要么是极低频可预测性(套利资金会迅速抹掉)。收益自相关不是量化的主战场 —— 波动率聚类才是。

2. GARCH:建模波动率聚类

核心思想:方差本身是时变的,且依赖过去的方差和冲击。Engle 1982 提出 ARCH, Bollerslev 1986 推广到 GARCH。

$$\text{GARCH}(1,1): \quad \sigma_t^2 = \omega + \alpha \varepsilon_{t-1}^2 + \beta \sigma_{t-1}^2$$

三项含义:

项含义
$\omega > 0$长期方差基线
$\alpha \varepsilon_{t-1}^2$昨天的大冲击 → 今天波动更大(ARCH 项)
$\beta \sigma_{t-1}^2$昨天的高波动 → 今天也高(GARCH 项,持续性)
GARCH 如何捕捉聚类

如果昨天有大冲击($\varepsilon_{t-1}^2$ 大),或昨天波动高($\sigma_{t-1}^2$ 大),今天 $\sigma_t^2$ 就大 —— 大波动跟着大波动。这就是 L02 看到的"波动率聚类"的数学表达。实证中 $\alpha + \beta \approx 0.99$(持续性极强),意味着波动率的"记忆"很长。

无条件方差:GARCH(1,1) 的长期方差 $\bar{\sigma}^2 = \omega / (1 - \alpha - \beta)$。要求 $\alpha + \beta < 1$(平稳性)。接近 1 意味着强持续、慢回归均值。

3. Python:用 arch 跑 GARCH

# pip install arch yfinance
import yfinance as yf
from arch import arch_model
import numpy as np

sp = yf.download("^GSPC", "2015-01-01", "2024-12-31")["Close"]
rets = 100 * np.log(sp / sp.shift(1)).dropna()  # 单位 %

# 拟合 GARCH(1,1) - 默认正态假设
am = arch_model(rets, mean="Constant", vol="GARCH", p=1, q=1)
res = am.fit(update_freq=0)
print(res.summary())

# 提取条件波动率(时变 σ_t)
cond_vol = res.conditional_volatility
print(f"\n平均条件波动率: {cond_vol.mean():.3f}%/日")
print(f"最高波动率时点: {cond_vol.idxmax()} ({cond_vol.max():.3f}%)")
典型结果
omega(ω) : 0.025  (p<0.01)
alpha(α) : 0.089  (p<0.01)
beta(β)  : 0.908  (p<0.01)
α + β    : 0.997  (强持续,慢回归均值)
解读

$\beta = 0.908$ 占主导 → 波动率主要靠"昨天的波动 + 一点冲击"决定,记忆长。$\alpha + \beta = 0.997$ 接近 1 → 波动率的"遗忘"很慢。危机期波动率高,会持续一段时间才回落。提取出的 cond_vol 就是时变波动率序列,比 L02 的固定窗口波动率更精确。

4. GARCH 的延伸:t 分布残差

L02 证明残差肥尾。GARCH 假设正态残差会低估尾部。改进:让残差服从 Student-t 分布:

am = arch_model(rets, mean="Constant", vol="GARCH", p=1, q=1, dist="t")
res = am.fit(update_freq=0)
# 看 t 分布自由度(越低 = 越肥尾)
GARCH(1,1) 中 $\alpha + \beta = 0.98$,含义是?
  • 残差是正态分布
  • 波动率恒定不变
  • 波动率有强持续性,慢回归均值
  • 模型错误,应丢弃
答案:C。$\alpha + \beta$ 接近 1 → 波动率的"记忆"很长(GARCH 平稳性要求 $\alpha + \beta < 1$,接近 1 意味着接近非平稳但还平稳)。这就是为什么危机期波动率高,会持续很久才回落 —— 这正是 L02 的"波动率聚类"。A 无关(残差分布另算);B 错($\alpha = \beta = 0$ 才恒定);D 错(0.98 < 1,模型合法)。
为什么金融收益通常 ARIMA(0,1,0) 就够,但波动率必须用 GARCH?
  • 收益无方差,波动率有方差
  • 收益无自相关,但方差有时变聚类
  • ARIMA 比 GARCH 难
  • GARCH 取代了 ARIMA
答案:B。弱有效市场 → 收益本身无自相关(ARIMA 退化为常数 + 白噪,即 (0,1,0));但方差有时变聚类(L02 实证过 $r^2$ 有强自相关),必须用 GARCH 建模。这就是为什么金融时间序列的标准模型常是 ARIMA(0,1,0) + GARCH(1,1) 组合。A 错(收益当然有方差);C/D 无关。

本课要点

  • ARIMA(p,d,q):建模均值自相关;金融收益通常 (0,1,0) 即可。
  • GARCH(1,1):$\sigma_t^2 = \omega + \alpha\varepsilon_{t-1}^2 + \beta\sigma_{t-1}^2$,刻画波动率聚类。
  • $\alpha + \beta$ 接近 1 = 波动率强持续,慢回归均值。
  • 提取 conditional_volatility 得到时变 $\sigma_t$,比固定窗口波动率精确。
  • GARCH + t 分布残差 = 同时处理聚类和肥尾,实证标配。
